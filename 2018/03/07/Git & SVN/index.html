Git & SVN

Veröffentlicht am 2018-03-07

前文

记录Git & SVN的相关

Git

1 如果创建工程的时候未将项目作为git源，我们需要做一下操作

cd /Users/hujianbo/Desktop/NoGit

切换到此工程

git init

这会初始化一个空的源，如果你在Finder里面查看或是输入ls命令，你会看到.git子目录已经被创建，很好，接下来输入：

git add .

这样，当前目录所有的内容就被添加到源里面去了，最后，输入以下命令：

git commit -m ‘Initial commit’

本地git源所执行的改变列表

重新打开工程

2 如果已经将此项目作为git源

在作了修改之后，点击sourceControl／commit

a 被修改的文件旁边出现字母m，代表此文件被修改，如果取消选中，则不会被修改

b 被修改的位置会出现数字，点击数字会有两个选项，一个是不提交修改，一个是忽略修改，忽略修改那么工程还是不会被改变。

c 在提交框下边可以输入提交备注。

3 版本比较

a 在view／version Editor下可以比较不同的版本

a1 点击双边编辑版

b 点击右边下面的时钟可以选择之前的任意版本与当前版本比较

b1 点击正下面的时钟可以看到版本提交次数

4 错误查找blame

a 点击View／Version Editor／Blame View／可以查找到每次修改的详细时间以及修改的内容

a1 点击 View／show log View也可以找到详细的修改日志

5 分支修改（不会影响稳定版本master）

a 点击source Control／工程名／new branch／产生一个新版本SecondBranch

b 修改SecondBranch不会影响master，然后提交。

c 点击source Control／switch master回到master版本，在source Control／merge from Branch，将需要合并的版本合并到master中。提交的时候工程下方有四个选项，选项对应代码的位置。

d 版本不提交不能重新合并。

6 删除修改 （不可恢复）

a1 选中被修改的文件，在source Control／disagard changes in select file，撤销所有选中文件的修改；disagreed all changes，删除所有修改。

a2 版本提交的时候，在被修改的位置点击修改的数字，disgard changes。

7 版本的远程push

a 注册gitHub账号，添加远程仓库

一

a 在终端某文件目录下输入 (本地仓库已经存在)

git remote add origin https://github.com/TooYongToFish/Test.git (添加远程push功能)

git push -u origin master （push代码或通过xcode自带的push）

二

a 直接通过xcode的第三项下载git项目，同时会自带远程push功能

三

a 在终端中某文件目录下输入

git init bare (添加远程push功能)

git clone https://github.com/TooYongToFish/Test （下载仓库代码）

8 回退到之前的版本

a 终端输入git reset –hard HEAD～2 (倒退版本数）

a1 终端输入 git revert HEAD (回到上一个版本)

注：这里的回退版本都是本地回退版本，一旦使用git pull又会和远程的进行merge，保持一致。

git pull

git stash pop
